# EasyCMS 测试数据准备单

## 说明

本清单用于给开发、后端、测试环境维护方准备稳定测试数据。

使用规则：

- 每条数据尽量对应明确的 `FUNC-xxx`
- 优先准备“稳定、可重复使用、不污染核心账号”的数据
- 能避免直接操作唯一管理员账号的场景，尽量不要复用 `admin`
- 若某条数据暂时无法提供，应明确标记阻塞影响的用例范围

字段说明：

- 数据项：需要准备的账号/角色/菜单/日志等
- 对应用例：关联的功能用例编号
- 用途：该数据用于验证什么
- 前置状态：创建时应具备的初始状态
- 是否必须：`必须 / 建议`
- 提供方：`后端 / 前端 / 测试 / 环境`
- 备注：风险、限制、补充说明

## 7. 当前最优先补齐

建议按以下顺序准备：

1. 账号：`user_locked`、`user_disabled`、`user_readonly`、`no_permission_user`、`vendor_admin`、`user_must_change_password`
2. 用户：启用/停用/锁定/重复用户名/多角色用户
3. 角色：预置角色、空角色、重复名称/编码角色、已绑定活跃用户角色
4. 菜单：稳定三级菜单树、目录节点、菜单项节点、按钮节点
5. 审计日志：失败日志、字段变更日志、登录类日志

## 8. 当前已知缺口

基于当前 `accounts.yaml`，已明确缺少：

- `user_readonly`
- `user_editor`
- `user_locked`
- `user_disabled`
- `vendor_admin`

这些缺口会直接影响权限矩阵、登录限制、状态流转、企业客户、日志范围等场景的自动化落地。
