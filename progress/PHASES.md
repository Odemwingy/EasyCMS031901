# 项目阶段与任务拆分

## 说明
本文件按阶段拆分项目任务，用于规划与跟踪。**每个阶段对应约 5 个工作日**。具体子任务与验收标准可在各阶段下补充，或与 `issue/` 关联。

与进度文件的配合方式：
- `CURRENT_PROGRESS.md` — **主控总览**：记录当前阶段 + 各端整体状态速览（链接到各阶段子目录）
- `阶段N/OVERVIEW.md` — **本阶段落地总览**：细致拆解本期所有任务，是开发者/AI 的直接执行依据
- `阶段N/frontend.md` — 本阶段前端领域详细进度（由前端开发者 / AI 维护）
- `阶段N/backend.md` — 本阶段后端领域详细进度（由后端开发者 / AI 维护）
- `阶段N/ui.md` — 本阶段 UI 设计领域详细进度（由设计师 / AI 维护）
- `阶段N/tests.md` — 本阶段测试领域详细进度（由测试工程师 / AI 维护）

> 并行开发时，各端**只写自己阶段子目录内的子进度文件**，避免多人同时修改同一文件产生冲突。`PHASES.md` 只在需求/阶段定义变更时更新；`OVERVIEW.md` 在任务拆解时生成，开发中可按需补充子任务。

## 阶段一（Day 1–5）：项目初始化 · 认证 · 布局 · 权限框架

- **目标**：搭建技术底座，完成登录认证与基础 RBAC 权限体系，实现整体布局框架与后台用户/角色管理入口。
- **技术选型**：Next.js 14（App Router）+ TypeScript + Prisma + PostgreSQL + TailwindCSS + shadcn/ui
- **交付物**：
  - 项目脚手架初始化（`app/` 下前后端目录结构）
  - 数据库 Schema 基础模型：User、Role、Permission
  - 登录 / 登出页面与 JWT 鉴权中间件
  - 整体布局：顶部导航栏（模块切换页签）、左侧一级侧边栏、面包屑、内容区
  - 后台管理 → 用户列表（增删改查）
  - 后台管理 → 角色列表与角色-权限配置页
  - 只读角色标识与基础权限拦截（页面级）

---

## 阶段二（Day 6–10）：内容中心基础 · 内容列表与详情骨架

- **目标**：完成内容中心的列表页与详情页骨架，支持视频、音频、游戏三类内容的新增、编辑、删除。
- **交付物**：
  - 数据库 Schema：Content（含基础字段：唯一标识、标题、内容类型/子类型、时长、年份、分级、语言、文件名、编码/加密方式、来源方、上下线时间）
  - 内容中心列表页：筛选栏（类型、状态、关键字）、排序、分页
  - 内容详情页 Tab 骨架（文件信息 / 系列·分集 / 其他信息 / 元数据 / 预告素材 / 图片素材 / 分类信息 / 适配信息 / 发布信息）
  - 文件信息 Tab：全部基础字段的展示与编辑
  - 内容新增弹窗 / 页面
  - 内容删除确认

---

## 阶段三（Day 11–15）：内容中心进阶 · 素材管理

- **目标**：完善内容详情中图片、字幕、音轨、预告素材管理，支持素材上传与必需图片标记。
- **交付物**：
  - 数据库 Schema：ContentImage、ContentSubtitle、ContentAudioTrack、ContentTrailer
  - 图片素材 Tab：支持海报图/横版图/竖版图/缩略图上传、预览、删除；必需图片标记
  - 字幕 Tab：多语言字幕新增、排序、加密标识、状态
  - 音轨 Tab：多语言音轨新增、排序、加密标识、差异化展示
  - 预告素材 Tab：预告片上传与关联管理
  - 系列·分集 Tab：系列关联与分集排序
  - 内容子类型配置（视频子类型 / 音频子类型）

---

## 阶段四（Day 16–20）：分类中心 · 分类树 · 分类集管理

- **目标**：实现分类树的层级展示与管理、分类集 CRUD，支持内容挂载到分类。
- **交付物**：
  - 数据库 Schema：CategorySet、CategoryNode（含全部节点属性字段）
  - 分类集列表页：新建、编辑、复制、另存为新、删除（未被引用时）、关联到周期/配置档
  - 分类树界面：层级展示、展开/收起、关键字检索、拖拽排序
  - 分类节点详情：父节点、名称、编码、排序、允许内容类型、锁定类型、虚拟分类、强制导出等全字段编辑
  - 内容挂载到分类（从内容详情「分类信息」Tab 操作）
  - 锁定类型约束逻辑（未锁定 / 部分锁定 / 完全锁定）

---

## 阶段五（Day 21–25）：状态清单 · 异常体系

- **目标**：完成状态清单页面，实现内容异常状态的枚举展示与筛选。
- **交付物**：
  - 状态清单页：按分类展示内容状态，含编排组关联 / 预告关联 / 图片 / 签核 / 版本 / 异常状态列
  - 异常状态枚举（12 种，见 PRD 8.2.6）及严重级别、是否阻断导出、修复建议、责任角色展示
  - 状态清单筛选：按内容类型、状态、分类、适配维度
  - 批量选中与批量挂类操作入口
  - 分类最小内容数规则配置与提示

---

## 阶段六（Day 26–30）：两阶段签核 · 变更申请

- **目标**：实现初始签核与全量签核的锁定机制，以及签核后变更申请的完整流程。
- **交付物**：
  - 数据库 Schema：SignOff（初始/全量）、ChangeRequest
  - 初始签核：技术字段锁定、签核操作、锁定状态展示与拦截
  - 全量签核：业务字段锁定、前置初始签核检查
  - 变更申请：提交 / 审批 / 驳回 / 取消流程
  - 变更申请列表（工作台与首页均可展示）
  - 同一来源修改归并为同一变更单
  - 变更通知提醒入口

---

## 阶段七（Day 31–35）：版本管理 · 周期管理

- **目标**：实现内容版本手动创建与查看，完成发布周期的创建、继承与状态管理。
- **交付物**：
  - 数据库 Schema：ContentVersion、Cycle
  - 版本管理：手动创建版本快照、版本差异查看、版本恢复、版本备注
  - 周期列表页：创建新周期 / 从上一周期继承 / 创建空周期
  - 周期详情：状态维护、更换分类集、关联配置档
  - 多周期并行展示
  - 周期锁定与受控修改入口

---

## 阶段八（Day 36–40）：导入与批量处理

- **目标**：实现内容对象的模板导入（支持覆盖/追加/更新/预检四种模式）、导入历史与批量操作。
- **交付物**：
  - 数据库 Schema：ImportJob、ImportLog
  - Excel/CSV 模板导入：视频、音频、游戏内容对象
  - 四种导入模式：覆盖 / 追加 / 更新 / 预检
  - 高风险覆盖二次确认提示
  - 导入历史列表（文件名、状态、时间、错误日志、操作者、模式）
  - 批量操作：批量挂类、批量更新字段、批量增加语言、批量激活发布目标

---

## 阶段九（Day 41–45）：完整性校验 · 发布导出

- **目标**：实现完整性校验引擎与导出任务的创建、前置校验、状态管理和导出摘要。
- **交付物**：
  - 完整性校验引擎（覆盖 PRD 8.9.1 全部 10 项校验范围）
  - 校验结果页：是否通过、问题列表、严重级别、修复建议、是否阻断导出
  - 变更检测：若无变更可提示跳过，有变更则强制重新校验
  - 导出任务创建与导出前校验（9 项，见 PRD 8.8.4）
  - 导出列表与导出历史
  - 导出摘要文件生成（含全部必需字段）
  - 导出保护：防止同一周期短时间重复导出
  - 导出状态：正常 / 警告 / 严重错误（严重错误禁止标记 Released/Testing）

## 阶段十（Day 46–50）：基础配置中心

- **目标**：实现配置档管理与核心配置对象（媒体配置、路由、分组结构、适配规则基础版本）。
- **交付物**：
  - 数据库 Schema：ConfigProfile、MediaConfig、Route、GroupStructure、AdaptationRule
  - 配置档列表：新建 / 编辑 / 复制 / 锁定 / 查看只读版本
  - 配置档详情：关联分类集、关联媒体配置、关联路由、关联分组、导出偏好、适配规则
  - 媒体配置管理：手工维护与基础字段
  - 路由管理：手工维护（名称/起点/终点/编号/关联媒体配置）
  - 分组/层级结构：舱位/区域/终端分组、显示值与导出值映射
  - 适配规则基础版本：机型 / 舱位 / 终端 / 区域四个维度；规则优先级（内容级 > 分类级 > 配置档级）
  - 媒体对象启用控制（按配置档维度）
  - 偏好设置：导出模式、加载类型、命名规则

---

## 阶段十一（Day 51–55）：工作台 · 基础报表 · 审计日志 · 通知

- **目标**：完成工作台总览、基础报表（媒体/变更/质量/操作）、审计日志与系统通知，收尾 MVP。
- **交付物**：
  - 工作台：左侧分类树快速入口 + 右侧周期里程碑 / 变更申请列表 / 待办任务 / 异常提醒
  - 工作台：组织/项目/年份/月份切换；上下文跟随当前周期与配置档
  - 内容服务商视角：仅展示分配给本组织的事项
  - 基础报表：媒体报表（筛选 + Excel 导出）、变更报表（原值/新值展示）、质量报表（状态聚合）、操作报表
  - 审计日志：内容编辑 / 签核 / 变更审批 / 配置修改 / 导入 / 导出日志，全量记录操作者/时间/对象/结果
  - 通知能力：导出结果 / 配置签核 / 变更申请 / 任务提醒（站内消息）
  - 整体联调、权限细化、Bug 修复
