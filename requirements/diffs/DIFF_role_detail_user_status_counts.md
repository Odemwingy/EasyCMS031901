# DIFF：角色详情增加启用/停用用户统计字段

## 1. 变更背景

- 触发原因：角色详情当前仅返回 `user_count`，前端无法直接区分启用用户数与停用用户数。
- 业务目标：增强角色详情信息密度，便于前端展示角色下用户状态分布。
- 生效范围：角色管理模块的角色详情接口 `GET /api/v1/admin/roles/{id}`。

## 2. 版本信息

- 从版本：当前阶段一执行基线
- 到版本：当前阶段一执行基线 + 本次角色详情统计增强
- 生效日期：2026-04-01
- 负责人：AI / 开发协作

## 3. 差异清单

### 一、增量

- 角色详情接口新增返回字段 `active_user_count`
- 角色详情接口新增返回字段 `disabled_user_count`
- 新增统计口径说明：
  - `active_user_count` 仅统计 `status = 1`
  - `disabled_user_count` 仅统计 `status = 2`
  - `status = 3`（锁定）与 `status = 4`（未激活）不计入上述两个新增字段

### 二、修改

- 角色详情接口响应示例由仅返回 `user_count`，调整为同时返回：
  - `user_count`
  - `active_user_count`
  - `disabled_user_count`
- 执行版接口基线文档已同步注明角色详情接口当前额外返回上述统计字段
- 后端角色详情资源返回结构已同步增强，但 `user_count` 原有口径保持不变

### 三、删除

- 无实质删除

## 4. 影响范围

- 前端影响：
  - 角色详情页可按需消费新增字段
  - 旧前端不读取新增字段时不受影响
- 后端影响：
  - 角色详情响应结构增强
  - 新增对应契约测试
- 数据模型 / 迁移影响：
  - 无数据库结构变更
  - 无历史数据迁移
- API 契约影响：
  - 属于响应字段追加
  - 不删除、不重命名现有字段
- 权限 / 审计影响：
  - 无新增权限点
  - 无新增审计日志要求

## 6. 验收标准

- [ ] 角色详情接口返回 `disabled_user_count`
- [ ] `user_count` 口径保持不变

## 7. 非目标

- 不调整角色列表接口
- 不新增锁定用户数或未激活用户数字段
- 不修改数据库 schema
- 不修改角色停用规则

## 8. 风险与回滚

- 风险点：
  - 前端可能将 `disabled_user_count` 误解为“非启用全部用户数”
  - 若后续产品要求增加锁定/未激活统计，需要再次明确字段口径
- 回滚策略：
  - 可仅回滚角色详情资源与对应测试
  - 文档同步回滚至仅保留 `user_count` 的版本
